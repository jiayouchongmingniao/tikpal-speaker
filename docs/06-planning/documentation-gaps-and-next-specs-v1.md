# 文档缺口与下一步规格清单 v1

## 1. 目标

这份文档用于盘点当前 `docs/` 中尚未明确写出的设计与工程规格，避免进入开发后才发现关键边界没有被定义。

---

## 2. 已覆盖的文档主题

当前已覆盖：

- Ambient OS 架构与模式切换
- 页面与组件边界
- 设备端服务与部署拓扑
- Overview / Listen / Flow / Screen 的主要产品规格
- 统一导航与手势规则
- System API Contract
- tikpal portable 控制器接入
- 树莓派 4 性能降级
- OTA 升级与回滚
- 实施路线、技术选型、风险清单
- 端到端用户场景脚本

---

## 3. 仍建议补充的高优先级文档

## 3.1 SystemState Schema 与字段字典 v1

为什么缺：

- 多份文档提到了 `SystemState`
- 但还没有一份专门的字段定义总表

建议内容：

- 完整字段
- 字段类型
- 字段来源
- 是否必须
- 哪些 UI 消费它

优先级：
- 高

---

## 3.2 Actions 语义字典与状态转换表 v1

为什么缺：

- 现在已经定义了不少 actions
- 但缺少一份系统动作总表和状态前后变化规则

建议内容：

- action 列表
- 输入源
- 前置条件
- 状态变化
- 失败条件

优先级：
- 高

---

## 3.3 ScreenContext 映射规则明细 v1

为什么缺：

- Screen 的产品结构已清晰
- 但 Calendar / Todoist 到 `ScreenContext` 的字段映射还不够细

建议内容：

- 当前任务优先级
- 下一项合并策略
- 事件 / 任务冲突规则
- stale / sync error 行为

优先级：
- 高

---

## 3.4 API 认证与权限模型 v1

为什么缺：

- 已经反复提到普通权限与管理权限
- 但没有正式写权限矩阵

建议内容：

- token 类型
- 会话生命周期
- 设备本机 / portable / admin 的权限差异
- OTA、connector、system action 的权限边界

优先级：
- 高

---

## 3.5 日志、遥测与调试面板规格 v1

为什么缺：

- 后续联调时会大量依赖日志
- 但尚未定义 action log / sync log / performance log 的结构

建议内容：

- 日志类型
- 必记字段
- 调试面板需要展示哪些实时状态
- 崩溃 / 降级 / 回滚记录方式

优先级：
- 高

- 总体架构判断
